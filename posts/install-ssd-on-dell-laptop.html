我把本子自带的 1T 机械盘换成了 120G 固态硬盘。

更换硬盘

先把笔记本的拆开，拆机的方法到网络上搜一下自己对应的型号，一般能搜出教程。 然后就是拆下原本的硬盘，并将 SSD 换上，这一步没有任何问题。

迁移系统

寻找解决方案

我需要一个迁移系统的办法。 在 万能的论坛 上找到了 @依云仙子 提供的方法。 

rsync -aviHAXKh --partial --delete src dst

该命令可以将 src 目录递归的拷贝到 dst 目录下，并且保留链接信息，也就是说软链接什么的不会丢失。

迁移过程

其实这个没有什么好说的，进入救援/备用系统1。挂载硬盘分区，然后使用 rsync 命令转移系统即可。 不过因为硬盘是 SATA 接口，所以我用了 USB 接口的移动硬盘作为中转，先将系统转移到移动硬盘中，在换上 SSD 再将移动硬盘里面的系统转移到 SSD 上。

重新配置分区和引导

原来的系统放在机械盘上，分区信息如下：

  • EFI -100M
  • 交换空间 -8G
  • 前 windows -200G
  • 家目录/home -else all
  • 根目录/ -200G

听说 SSD 不宜分太多区，所以我决定就用了一个根分区/ 因为磁盘类型还是用的 gpt，所以需要一个 efi 分区，所以最后 SSD 分区如下：

  • efi -100M
  • / -else all

因为分区信息不一样了，所以需要重新配置/etc/fstab 文件， 另外我还重新配置了一下 grub 程序，不过似乎不是很必要，因为我的磁盘标识都是/dev/sda。
